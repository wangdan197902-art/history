# 地区化"今天历史上发生了什么"档案站 — 可行性评估报告

> 评估日期：2026-07-18
> 评估维度：现实可行性、实施简易度、成本效益、目标契合度、创新性
> 任务类型：探索型（权重：可行性15%+简易度10%+成本15%+契合度35%+创新25%）

## 一、题材基本信息

- **题材名称**：地区化"今天历史上发生了什么"档案站
- **一句话定位**：每个国家×每天=1个页面，"on this day in {country} history"，覆盖150国×365天=5.4万页
- **类型**：数据型 + 内容型
- **目标受众**：教育工作者、学生、历史爱好者、旅行者、海外侨民、各国本地用户
- **目标语言**：英文+5种小语种（西/法/葡/阿/俄）

## 二、市场分析

### 2.1 市场容量与趋势

- **市场容量**：全球历史教育内容搜索月均5亿+次；"on this day"类查询英语月2000万次，小语种月1亿次+
- **趋势红利**：
  - Wikipedia"OnThisDay"垄断英语，但小语种+具体国家空白
  - 教育市场CPC稳定，广告主充足
  - 每日更新=天然回访机制（用户每日查看当日历史）
  - 邮件订阅"每日一国历史"是高留存渠道
- **市场天花板**：150国×365天×5语种=27万页面，理论月50-200万UV

### 2.3 预估CPC与RPM

- **CPC区间**：$0.8-2.5（教育市场CPC中等）
- **RPM预估**：$4-12（教育+旅行联盟）
- **联盟转化率**：在线课程/旅行预订2-5%

### 2.4 竞争强度分析

- **英语竞争**：高（Wikipedia"OnThisDay"+History.com+OnThisDay.com垄断）
- **小语种竞争**：低（小语种+具体国家页面稀缺）
- **本地化竞争**：中（部分国家有本地历史网站，但SEO和技术弱）
- **总体竞争强度**：低（小语种+具体国家双维度蓝海）

### 2.5 差异化机会

- **150国×365天矩阵覆盖**：唯一全球+本地化历史档案站
- **每日更新机制**：天然回访+邮件订阅留存
- **AI叙事改写**：避免抄袭Wikipedia，AI改写为故事化叙事
- **5语种同步**：每国历史×5语种=5个SEO页面
- **关联推荐**：同日不同国历史互链，跨国历史教育

## 三、技术可行性

- **静态站点生成器**：Hugo（首选，超快构建5万+页）/ Astro（备选）

## 六、竞争分析

### 6.1 直接竞品（3-5个）

| 竞品 | 优势 | 办势 |
|------|------|------|
| Wikipedia OnThisDay | 权威性高、SEO权重高 | 仅英语、无国家细分、无叙事 |
| History.com | 英语权威 | 仅英语、无多语言、广告重 |
| OnThisDay.com | 老站权重高 | 仅英语、内容浅、无国家维度 |
| 各国小语种历史网站 | 本地权威 | SEO弱、技术老旧、覆盖少 |
| 各国本地新闻"今日历史"栏目 | 时效性好 | 不系统、SEO弱 |

### 6.2 间接竞品

- **Wikipedia各国语言版**：每国本地历史
  - 对策：跨国整合+5语种统一覆盖
- **AI搜索直接回答**：简单事件被AI截流
  - 对策：深度叙事+关联推荐+多国视角
- **历史纪录片/视频**：视频内容
  - 对策：文字快速查阅+SEO优势

### 6.3 竞争壁垒构建

- **覆盖广度壁垒**：150国×5语种=750个本地化版本
- **数据深度壁垒**：5.4万国×日页面，3年累积后难以追平
- **SEO壁垒**：每国每日页面互链，形成历史知识网络
- **回访机制壁垒**：每日更新+邮件订阅，用户留存高
- **多语种壁垒**：5语种同步覆盖，本地化竞品难追

- **风险点**：教育内容天然合规

  - 各国官方历史网站遵守ToS
  - 历史事实无版权

## 八、AI参与的完整闭环运行流程（核心必填）

### 8.1 闭环架构图

```
[选题AI(固定150国×365天)] → [数据抓取(Wikipedia API)] → [生成AI(Claude改写+Gemini翻译)] 
   ↓                                                              ↓
[反馈优化] ← [监控AI(GSC+GA4)] ← [SEO AI(JSON-LD)] ← [发布AI(每日cron)]
   ↓
[当年事件补充]
```

### 8.2 选题模块

- **触发条件**：每日UTC 01:00（cron: `0 1 * * *`）
- **数据源**：
  - Wikipedia OnThisDay API：`GET https://en.wikipedia.org/api/rest_v1/feed/onthisday/events/{MM}/{DD}`
  - 各国Wikipedia语言版OnThisDay API
  - DBpedia SPARQL查询
- **AI工具**：Claude 3.5 Sonnet判定事件与各国相关性
- **Prompt模板**：
  ```
  你是历史编辑。以下是{date}的全球历史事件JSON：
  {events_json}
  
  请为以下国家筛选相关事件：{country_list}
  
  每国筛选标准：
  1. 事件发生在该国境内
  2. 事件涉及该国公民/政府
  3. 事件对该国有重大影响
  
  输出JSON：{"country_code": ["event1", "event2", ...]}
  ```
- **输出**：`/data/queue/today_events_{YYYYMMDD}.json`

### 8.3 生成模块

- **触发条件**：选题完成（每日02:00）
- **AI工具**：
  - Claude 3.5 Sonnet：改写为叙事+生成侧栏
  - Gemini 1.5 Flash：翻译5语种
  - GPT-4o：生成SEO标题+meta
- **Prompt模板（叙事改写）**：
  ```
  你是历史编辑。为{country}生成"on this day in {country} history"页面。
  
  相关事件：
  {events}
  
  输出：
  1. 每事件改写为50-150字叙事（非Wikipedia原文）
  2. 按重要性排序
  3. 加"今日相关人物"侧栏（3-5人）
  4. 加"今日相关地点"侧栏（3-5地点）
  5. 加"跨国关联"（与其他国家同日事件联系）
  6. SEO meta title + description
  
  风格：客观、叙事化、教育性
  ```
- **Prompt模板（翻译）**：
  ```
  将以下"on this day in {country} history"页面翻译为{target_language}：
  {content}
  
  要求：
  1. 人名/地名按本地习惯翻译
  2. 历史事件表述符合本地认知
  3. SEO标题符合本地搜索习惯
  ```
- **输出**：`/content/{lang}/{country}/{MM-DD}.md`

### 8.4 审核模块

- **触发条件**：内容生成完成
- **AI工具**：
  - GPT-4o：原创性检测（vs Wikipedia）+ 敏感政治事件识别
  - 自动化脚本：事件数量校验+链接有效性
- **人工环节**：
  - 抽检比例：5%（主要国家每日抽检）
  - 标准：原创性、政治中立性、SEO合规
  - 审核日志：`/data/audit/log_YYYYMMDD.json`
- **输出**：审核报告+通过/打回标签

### 8.5 发布模块

- **触发条件**：审核通过（每日03:00）
- **CI/CD**：GitHub Actions workflow
  ```yaml
  name: Daily History Update
  on:
    schedule:
      - cron: '0 1 * * *'  # UTC 1点触发
    push:
      branches: [main]
      paths: ['content/**']
  jobs:
    generate-and-deploy:
      runs-on: ubuntu-latest
      steps:
        - uses: actions/checkout@v4
        - name: Generate today's pages
          run: python scripts/generate_daily.py
        - name: Commit and push
          run: |
            git add content/
            git commit -m "Daily update $(date -u +%Y-%m-%d)"
            git push
        - uses: hugo-action@v1
        - uses: cloudflare/pages-action@v1
          with:
            apiToken: ${{ secrets.CF_API_TOKEN }}
            accountId: ${{ secrets.CF_ACCOUNT_ID }}
            projectName: on-this-day
  ```
- **部署**：Cloudflare Pages自动构建（5万+页约2-5分钟）
- **邮件订阅**：自动发送当日历史邮件（Buttondown API）
- **输出**：线上页面+sitemap更新+邮件推送

### 8.6 SEO/AEO模块

- **自动生成**：
  - meta标签：AI生成（含国家+日期+事件关键词）
  - JSON-LD：Article schema + Event schema
    ```json
    {
      "@type": "Article",
      "headline": "On This Day in Brazil History: July 15",
      "datePublished": "2026-07-15",
      "about": {"@type": "Event", "name": "..."}
    }
    ```
  - sitemap.xml：自动生成，按国家分
  - hreflang标签：5语种互链
  - llms.txt：AI Agent可读索引
- **提交**：Google Search Console API自动提交当日URL
- **AEO优化**：
  - 事件结构化数据
  - 时间线视图（AI搜索偏好）
- **输出**：收录状态+AI引用监控

### 8.7 监控反馈模块

- **触发条件**：每日UTC 23:00
- **数据源**：
  - Google Search Console API
  - Google Analytics 4
  - 邮件订阅后台（Buttondown）
- **AI分析**：GPT-4o分析
  - 流量趋势：哪些国家/日期页流量高
  - 邮件订阅留存：哪些国家历史订阅留存高
  - 内容质量：跳出率>80%的页面识别
  - 迭代建议：哪些国家需要更深度内容
- **输出**：`/data/optimization/daily_report.json` → 反馈选题模块
