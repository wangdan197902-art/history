## 1. 业务定位与价值主张

### 1.1 一句话定位

**每天 × 地区 = 1 个页面**，记录该地区当天发生的历史事件，做"地区化的 Wikipedia On This Day"。

### 1.2 详细定位

| 维度 | 描述 |
|------|------|
| 内容品类 | 历史 / 教育 / 文化 / 资讯参考 |
| 内容形态 | 每日更新的结构化档案（事件 + 配图 + 引用） |
| 内容深度 | 中等深度（每事件 50-300 字叙事 + 多源引用） |
| 内容广度 | 30 地区 × 366 天 × 10 语种 ≈ 10 万页面 |
| 更新节奏 | 每日新增 300+ 页面，年度全量刷新 |
| 主要语言 | 英语 / 中文 / 西班牙语 / 法语 / 德语 / 意大利语 / 葡萄牙语 / 俄语 / 日语 / 韩语 |
| 用户访问模式 | 周期性回访（每日 / 每月同日 / 每年同日） |

### 1.3 目标用户画像

| 用户类型 | 占比预估 | 核心诉求 | 典型场景 | 关键词示例 |
|---------|---------|---------|---------|-----------|
| 历史爱好者 | 30% | 拓展知识，发现冷门事件 | 每日浏览本国+他国历史 | "today in history France" |
| 学生 | 20% | 作业、考试、论文素材 | 搜索特定日期事件 | "what happened July 14 1789" |
| 教师 | 10% | 课堂素材、每日一题 | 课前 5 分钟展示 | "on this day lesson plan" |
| 媒体从业者 | 10% | 资料查证、选题灵感 | 节日/纪念日策划 | "Berlin Wall fall date" |
| 海外侨民 | 10% | 思乡情感、母语阅读 | 查看祖国当日历史 | " Filipino history June 12" |
| 旅行者 | 5% | 行前文化预习 | 搜索目的地历史 | "Mexico history September 16" |
| 地区文化爱好者 | 10% | 比较研究、跨国视角 | 同日多国对比 | "today in history multiple countries" |
| 学术研究者 | 5% | 引用素材、事实核查 | 长期查阅 | "Berlin Conference 1884 sources" |

### 1.4 价值主张

**对用户：**
- **唯一性**：全球唯一覆盖 30 地区 × 10 语种 × 366 天的统一档案
- **效率性**：1 个 URL 看完某国当日所有重大历史事件
- **中立性**：AI + 人工双轨审核，多源交叉验证
- **可读性**：叙事化改写（非 Wikipedia 原文），易读易懂
- **多语种**：母语阅读，非英语用户也能获取完整内容

**对合作伙伴（媒体/教育）：**
- **API 接入**：可付费嵌入到媒体每日"今天历史"栏目
- **白标授权**：教育平台可定制品牌版本
- **PDF 年鉴**：按需印刷，作为教学资料

**对广告主：**
- **教育垂直流量**：CPC $1-5，高质量用户
- **地域精准**：30 地区独立页面，地域定向
- **语言精准**：10 语种，语言定向

### 1.5 差异化优势

| 维度 | 本项目 | Wikipedia OnThisDay | History.com | 各国本地历史站 |
|------|--------|---------------------|-------------|----------------|
| 地区维度 | ✅ 30 地区独立页面 | ❌ 仅全球合并 | ❌ 仅英语 | ✅ 单一地区 |
| 多语种 | ✅ 10 语种 | 部分（按语言版本） | ❌ 仅英语 | ❌ 单语 |
| 叙事改写 | ✅ AI 叙事化 | ❌ 词条式 | ✅ 长文 | 部分 |
| 每日更新 | ✅ 每日 300+ 页 | ✅ | ❌ 不定期 | 部分 |
| 邮件订阅 | ✅ 每日推送 | ❌ | ❌ | ❌ |
| 关联推荐 | ✅ 同日他国 | ❌ | ❌ | ❌ |
| 引用规范 | ✅ 多源+CC BY-SA | ✅ | ✅ | 部分 |
| 中立性评分 | ✅ 0-1 量化 | 部分 | ✅ | 部分 |

### 3.2 核心功能清单

#### 3.2.1 日期浏览（Today）

**功能描述：** 用户访问首页，自动展示当日 30 地区的"今天历史"摘要卡片。

**用户故事：**
- 作为历史爱好者，我希望访问首页就能看到今日所有地区的历史摘要，以便快速浏览
- 作为海外侨民，我希望可以快速定位到我的祖国当日历史
- 作为教师，我希望可以一键复制当日事件链接到课堂演示

**功能规格：**
- 首页展示 30 地区卡片（按访问量排序，默认美国/英国/德国排前）
- 每张卡片显示：国旗 + 国名 + 当日 Top 3 事件标题 + "查看全部"按钮
- 自动根据浏览器语言（Accept-Language）展示对应语种首页
- 移动端单列瀑布流，桌面端 3 列网格

#### 3.2.2 地区筛选（Countries）

**功能描述：** 用户可按地区筛选历史事件。

**功能规格：**
- 地区列表页：30 国旗 + 名称网格展示
- 点击进入地区主页 `/en/on-this-day/us/`，展示该国：
  - 今日历史事件
  - 该国所有月份导航
  - 该国热门日期 Top 10（按历史访问量）
- 支持按事件类型筛选：政治/军事/科学/文化/体育/灾难/经济/宗教/法律/社会

#### 3.2.3 事件搜索（Search）

**功能描述：** 全文搜索 10 万页历史事件。

**功能规格：**
- 基于 Pagefind 静态搜索
- 支持关键词、年份范围、地区、事件类型组合筛选
- 搜索结果按相关性 + 时间排序
- 搜索页面 URL：`/search/?q={query}&country={code}&year={YYYY}&type={type}`
- 搜索结果展示：事件标题 + 年份 + 地区 + 摘要前 200 字

#### 3.2.4 日历视图（Calendar）

**功能描述：** 12 个月日历视图，每个日期可点击查看该日所有地区事件。

**功能规格：**
- 日历页 `/calendar/` 展示当年 12 个月
- 每个日期显示：事件总数 + 高亮（重大事件红色，普通事件灰色）
- 点击日期跳转到 `/on-this-day/all/{MM-DD}/`（所有地区汇总）
- 支持切换年份查看（仅展示有内容的日期）

#### 3.2.5 邮件订阅（Subscribe）

**功能描述：** 每日发送当日历史邮件。

**功能规格：**
- 订阅表单：邮箱 + 选择订阅地区（多选，最多 5 个）
- 双重 opt-in 确认（GDPR 合规）
- 每日 UTC 06:00 发送（按用户时区延迟发送）
- 邮件内容：5 个事件摘要 + "查看完整版"链接 + 退订链接
- 退订链接每封邮件必含
- 订阅偏好页：可随时修改订阅地区、邮箱、频率

#### 3.2.6 多语种切换（Language Switch）

**功能描述：** 用户可在 10 语种之间切换。

**功能规格：**
- 页面右上角语言选择器（下拉菜单）
- 切换后跳转到对应语种当前页（hreflang 互链）
- 自动检测浏览器语言，首次访问跳转（仅一次，避免循环）
- Cookie 记忆用户偏好（合规：仅必要 cookie）

#### 3.2.7 引用溯源（Sources）

**功能描述：** 每个事件显示引用来源。

**功能规格：**
- 每个事件底部显示：来源 1 + 来源 2 + 检索日期
- 引用来源链接到 Wikipedia / 国家档案馆 / 学术数据库
- 引用页 `/sources/` 说明引用规范（CC BY-SA 4.0）

#### 3.2.8 AI 内容披露（Disclosure）

**功能描述：** 每个 AI 生成页面披露 AI 参与情况。

**功能规格：**
- 每页底部固定显示："本内容由 AI 辅助生成，人工编辑审核。"
- 披露页 `/ai-disclosure/` 说明：
  - AI 模型版本（Claude 3.5 / GPT-4o / Gemini 1.5）
  - 人工审核流程
  - 用户反馈渠道
- 满足 FTC AI 内容披露要求

### 3.3 API 接口（开发中）

**v1 API 端点：**

```
GET /api/v1/events/{MM-DD}
```

## 8. 合规与风控架构

### 8.1 内容版权合规

| 内容类型 | 许可证 | 合规要求 |
|---------|--------|---------|
| Wikipedia 文本 | CC BY-SA 4.0 | AI 改写为原创叙事 + 标注来源 |
| Wikipedia 数据 | CC BY-SA 4.0 | 事实数据无版权，引用规范 |
| Wikimedia 图片 | 公共领域 / CC BY-SA | 每张图片标注许可证 + 作者 |
| AI 生成文本 | 项目所有 | 人工审核 + AI 披露 |
| AI 生成图片 | 项目所有 | 标注 "AI-generated" |
| 国家档案馆 | 各自许可 | 邮件申请授权 |

### 8.2 隐私合规

**GDPR（欧盟）：**
- 邮件订阅双重 opt-in
- 隐私政策明确数据用途
- Cookie consent banner（仅必要 cookie 默认开启）
- 用户数据删除请求 API
- 数据处理记录（DPA）

**CCPA（加州）：**
- "Do Not Sell My Personal Information"链接
- 用户数据查询 / 删除请求
- 隐私政策披露数据收集范围

**儿童隐私（COPPA）：**
- 不主动收集 13 岁以下儿童信息
- 教育版不收集任何学生数据

### 8.3 广告合规

**AdSense 政策：**
- 内容原创性（AI 改写 + 人工审核）
- 不发布敏感内容（暴力 / 仇恨 / 色情）
- 不发布受制裁内容（纳粹象征等地区法规）
- 充分的"About"和"Contact"页面
- AI 内容披露

**FTC 披露：**
- Amazon 联盟链接标注 "#ad" 或 "As an Amazon Associate I earn from qualifying purchases"
- 赞助内容标注 "Sponsored"
- AI 生成内容标注

### 8.4 敏感事件风控

**敏感事件类型与处理：**

| 事件类型 | 风险等级 | 处理流程 |
|---------|---------|---------|
| 纳粹 / 大屠杀（德国） | 高 | 必须人工审核 + 引用权威源 |
| 亚美尼亚事件（土耳其） | 高 | 必须人工审核 + 双方视角 |
| 俄乌冲突（俄罗斯/乌克兰） | 高 | 必须人工审核 + 中立表述 |
| 朝鲜战争（朝鲜半岛） | 中 | 标准审核 + 中立表述 |
| 文革 / 六四（中国） | 高 | 必须人工审核 + 多源引用 |
| 殖民地历史（多国） | 中 | 标准审核 + 双方视角 |
| 宗教冲突 | 中 | 标准审核 + 尊重原则 |
| 领土争议 | 中 | 标准审核 + 中立表述 |

**敏感事件处理 SOP：**
1. AI 自动检测敏感关键词 → 标记 `sensitivity_flags`
2. AI 审核模块评估中立性评分（< 0.7 必须人工）
3. 人工审核：编辑 + 历史顾问双签
4. 多源交叉验证（至少 3 个独立来源）
5. 发布后监控用户反馈，必要时下架修订

### 12.5 退出机制

**积极退出（被收购）：**
- 触发条件：月收入 > $10,000 + 月 UV > 50 万
- 潜在买家：教育集团（如 Coursera）/ 媒体集团（如 History.com）/ 历史出版社
- 估值模型：3-5 倍年收入 + 数据资产价值

**消极退出（项目终止）：**
- 触发条件：连续 6 个月亏损 + 月 UV < 1,000
- 处理方式：
  - 数据开源到 GitHub（CC BY-SA）
  - 域名出售
  - 邮件订阅用户移交流转
  - 财务清算

**转型退出：**
- 触发条件：核心团队精力转移
- 处理方式：
  - 寻找运营接班人
  - 内容授权给媒体合作伙伴
  - 维持最低运营（每日管道继续，停止主动运营）
