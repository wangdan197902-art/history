## 一、登记原则

- **显式登记**: 所有妥协、占位、延后实现的内容必须登记
- **可追溯**: 每项债务记录引入阶段、原因、还款计划
- **可量化**: 标注影响范围和还款优先级

## 二、债务清单

### TD-001: Mock 数据无法 100% 代表真实 API
- **引入阶段**: Phase 0
- **原因**: 用户无法访问国外网络,Mock 数据为人工构造
- **影响**: 真实 API 字段格式可能与 Mock 不一致,生产迁移时可能失败
- **还款计划**: Phase 4 用 VCR.py 录制真实响应对比
- **优先级**: 高

### TD-002: 6 个 Real Provider 实现为占位
- **引入阶段**: Phase 1
- **原因**: Provider 接口需先定义,Real 实现延后到 Phase 4
- **影响**: Phase 0-3 仅能跑 Mock,无法验证真实 API 集成
- **还款计划**: Phase 4 启用 Real Provider 实现
- **优先级**: 中

### TD-003: Hugo 构建策略取决于硬件评估结果
- **引入阶段**: Phase 0
- **原因**: 10 万页面构建可能 OOM,需根据 Mac 内存决定策略
- **影响**: 8GB Mac 可能无法全量构建
- **还款计划**: 运行 `python scripts/check_hardware.py` 后填入实际策略
- **优先级**: 高

### TD-004: Mock fixture 体积约 200MB
- **引入阶段**: Phase 0
- **原因**: 10,980 个 JSON 文件,每文件约 18KB
- **影响**: Git 仓库膨胀
- **还款计划**: 加入 .gitignore(可选) 或使用 git-lfs
- **优先级**: 低

### TD-005: schemathesis 契约测试需 Mock Server 启动
- **引入阶段**: Phase 0
- **原因**: 契约测试需运行中的 HTTP 服务
- **影响**: Phase 0 验收需手动启动 Mock Server
- **还款计划**: Phase 1 Mock Server 完成后自动启动
- **优先级**: 中

### TD-006: 故障注入测试默认关闭
- **引入阶段**: Phase 1
- **原因**: 故障注入中间件实现复杂,默认关闭降低复杂度
- **影响**: 无法主动测试管道容错能力
- **还款计划**: Phase 2 完成后按需启用 `FAULT_INJECTION_ENABLED=true`
- **优先级**: 中

### TD-007: Pagefind 索引体积未优化
- **引入阶段**: Phase 3
- **原因**: 10 万页索引可能 > 100MB
- **影响**: 用户首次搜索下载量大
- **还款计划**: Phase 3 完成后评估是否分片
- **优先级**: 低

### TD-008: 监控告警仅控制台日志
- **引入阶段**: Phase 3
- **原因**: Slack Webhook 需要 Phase 4 配置
- **影响**: 长时间运行管道无实时告警
- **还款计划**: Phase 4 配置 Slack Webhook
- **优先级**: 低
