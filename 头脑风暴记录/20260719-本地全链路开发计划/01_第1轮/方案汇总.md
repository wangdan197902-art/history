### 汇总方案 5：Docker Compose 环境对等方案（备选，不推荐）

**整合来源**：

- 技术实现者方案3（Docker Compose 一键环境）
- 战略架构师方案3（本地镜像生产环境）

**核心特点**：

- 8 个容器：mock-wikipedia / mock-anthropic / mock-openai / mock-gemini / local-gitea / local-minio / local-nginx / local-mailhog
- 环境对等：本地与生产架构一致
- 端点切换：`*_API_BASE` 环境变量

**不推荐理由**：

1. Docker Desktop 在 Mac 上资源占用大（建议 16GB+）
2. volume 性能损耗导致 Hugo 构建慢 2-3 倍
3. Gitea Actions 与 GitHub Actions 兼容性风险
4. 容器编排复杂度高，维护成本大

**适用场景**：仅当用户本地具备 32GB+ 内存且有强烈"环境一致性"需求时考虑。

### 汇总方案 6：VCR.py 生产数据回放方案（延后到生产迁移阶段）

**整合来源**：测试架构师方案2

**核心特点**：

- 首次运行时用 VCR.py 录制真实 API 响应
- 后续运行直接回放 cassette
- Mock 数据 100% 真实

**延后理由**：

1. 首次录制需网络访问，不符合用户当前约束
2. cassette 维护成本高（API 升级需重新录制）

**适用场景**：生产迁移阶段，用 VCR.py 录制真实响应，对比 Mock 差异，作为生产验证手段。
