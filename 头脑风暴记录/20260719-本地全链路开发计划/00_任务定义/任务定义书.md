# 任务定义书 — 本地全链路开发计划

> 创建时间：2026-07-19
> 执行级别：L3（深度模式）
> 任务类型：执行型

---

## 1. 任务描述

为"地区化今天历史档案站"项目制定**完整的开发计划**，输出要求：

1. **完整的开发步骤**：覆盖从环境搭建到上线运行的整个开发周期
2. **每步骤独立成文件**：每个步骤对应独立的 `.md` 文件，详细写明具体开发任务内容（含目标、前置条件、任务清单、验收标准、产出物）
3. **本地测试环境（全链路模拟）**：用户**无法访问国外网站**（Wikipedia API、OpenAI/Anthropic/Gemini API、Cloudflare Pages 均不可达），需先在本地完整跑通流程，再迁移到生产环境

## 2. 任务背景

### 2.1 项目概况

- **项目**：地区化"今天历史"档案站
- **目标规模**：366 天 × 30 地区 × 10 语种 ≈ 10 万页面静态站
- **技术栈**：Hugo + Cloudflare Pages + GitHub Actions + AI 管道（Claude/GPT-4o/Gemini）
- **已有文档**：8 份方案文档（技术架构、业务架构、内容生产闭环、SEO、部署运维、路线图、风险合规、AI 闭环）

### 2.2 关键约束

| 约束 | 说明 | 影响 |
|------|------|------|
| **网络隔离** | 用户无法访问 Wikipedia、OpenAI、Anthropic、Google、Cloudflare API | 必须本地 Mock 全部外部依赖 |
| **先走通本地** | 必须先在本地完整跑通流程，再迁移到生产 | 开发计划需明确"本地版"和"生产版"两套配置 |
| **看到网站内容** | 用户最终要看到真实可浏览的网站 | 本地 Hugo server 必须能渲染真实数据 |
| **每步独立文件** | 步骤文档需独立、详细、可执行 | 文档结构需标准化 |
| **最小化修改** | 优先独立新代码包，不侵入现有文档 | 新增 `开发计划/` 目录 |

### 2.3 关键技术挑战

1. **Wikipedia API 本地化**：需要本地 Mock 数据（事件池 JSON）
2. **AI API 本地化**：需要 Mock Claude/GPT-4o/Gemini 响应（或使用本地开源模型）
3. **Cloudflare Pages 本地化**：使用 Hugo server + 本地预览替代
4. **GitHub Actions 本地化**：使用本地脚本/Makefile 模拟 cron 调度
5. **30 地区 × 10 语种数据生成**：本地需有足够样本数据验证全流程

## 3. 关注维度

- **完整性**：开发步骤覆盖所有模块（前端/管道/数据/部署/监控）
- **文件独立性**：每个步骤文档独立可读，不依赖其他文档上下文
- **本地模拟真实性**：Mock 数据需足够真实以验证模板和管道
- **全链路可走通**：本地一次执行可走完 fetch → regionalize → translate → illustrate → audit → publish → build → preview
- **可迁移性**：本地版本可平滑迁移到生产环境（仅改配置不改代码）

| 项目 | 内容 |

| 角色 | 来源域 | 入选理由 |

| 性能工程师 | 开发型-扩展 | 10 万页 Hugo 构建、本地构建性能优化，确保大规模页面可行 |
